#config #配置
# 联网
首先启用`iwd`服务和`systemd-resolved`服务。
```bash
systemctl enable --now iwd
systemctl enable --now systemd-resolved
```
再按照[iwd教程](https://wiki.archlinuxcn.org/wiki/Iwd)里面讲的重新配置一下网络，并且仔细看完[这个](https://wiki.archlinuxcn.org/wiki/Iwd#%E5%90%AF%E7%94%A8%E5%86%85%E7%BD%AE%E7%BD%91%E7%BB%9C%E9%85%8D%E7%BD%AE)去配置`DHCP`。
> [!warning] 注意
> 注意所有的“配置完成但是不生效”，请先关注服务**是否启动或者重启**！

## 构建AUR
### 使用提示
不要在`root`下使用`git clone`,因为等下在`makepkg`的时候只能用普通用户，但是下载下来的文件是`root`的，普通用户没有访问权限。
### 好多依赖问题，我还要一个一个下载好累！
在`makepkg`后面加上`-s`参数就不用自己下载依赖了。
> [!note] 吐槽
> 感觉arch的干净感只在没有安装这些文件的时候才有的。
### 安装
安装：`makepkg -i`
注意`sudo`是一个包，需要`pacman`的。
### 聪明的你一定会使用yay的，对吧？
```bash
git clone https://aur.archlinux.org/yay.git
cd yay
makepkg -si
```
### 脚本自己从github下载太慢啦！
直接找镜像下载好了之后扔到对应的**安装文件夹的根目录**里面，然后再启动脚本就可以了。
### 下载失败之后就一直下载不了了！即使调整梯子也不行！
大概率是因为github不能断点续传，要手动把目录下的`*.part`删掉才行。
> [!hint] 猜测
> 可能是`range`请求爆了。

```bash
rm *.part
```
